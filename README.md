# SwordCatch👏
## ゲーム説明👾
このゲームは白刃取りのミニゲームです。🗡<br>
白刃取りの練習をして、白刃取りマスターになりましょう！

### ⚠️ 注意点

> [!IMPORTANT]
> ゲーム音が聞こえる必要があります。

<details>
<summary>コーディングで苦労・意識した部分</summary>
<br>
  
- 苦労した部分

このプロジェクトは本来は白刃取りのようなゲームがいくつもある、<br>
MiSideのようなゲームデザインのゲームを作るつもりだったので、<br>
スクリプトも様々なゲームの状況に対応できるように作っていたのですが、<br>
多層構造のStatePattern、それに付随するFactoryPatternなど設計が複雑になっていき、<br>
次第に処理を把握するのがとても難しくなったことにはとても苦労しました。<br>
また当時は汎用性や拡張性を優先しすぎるあまり、過剰にクラスを分け複雑な構造を乱立する傾向にあり<br>
逆に様々なクラスが密に関わる設計を作ってしまっていたので、密な設計故の不自由さに苦悩しました。<br>

- 意識した部分

意識した部分は制作当時は拡張性・汎用性です。<br>
しかし数カ月たってから自身のコードを見返してみると、<br>
とても読めたものではなかったので、2026年6月辺りからリファクタリングを少しづつ始めました。<br>
その際に意識した部分は、シンプルだけど拡張性や汎用性が確保されているコードです。<br>
その設計思想に至ったのは現場のエンジニアの方に<br>
「C＃のListは内部では複雑な事をしているけど、とても使いやすいようになっている、<br>
難しいコードが理解できる人はそういうことをする必要がある。」というアドバイスを頂き、<br>
Unityのクラスなどを見ても簡単に使えるようになっていると感じ、<br>
シンプルな設計にこそ巧さが表れると考えるようになったことがきっかけです。
</details>
